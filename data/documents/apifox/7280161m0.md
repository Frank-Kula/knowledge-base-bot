# 接口请求中 Header 携带 Authorization 参数不生效问题

Apifox 中 Authorization 参数未正确带入请求头，可能有以下几种原因：

1. 检查接口及父级目录的 Auth 设置： Apifox 中，接口和父级目录都可以设置 Auth 信息。请检查接口、父级目录、根目录是否设置了与 Header 同名的 Authorization 参数。Auth 的优先级高于 Header 中单独设置的参数，如果在 Auth 中设置了 Authorization，则 Header 中的设置会被覆盖。

2. 检查变量的使用： 如果你使用了变量来存储 Authorization 的值，请确保变量名拼写正确，并且该变量在当前环境中已赋值。可以查看[实际请求](https://docs.apifox.com/actual-request.md)，确认 Authorization 的值是否正确。

3. 前置脚本的影响： 检查[前置脚本](https://docs.apifox.com/pre-request-scripts.md)中是否有修改或删除 Authorization 的代码。
