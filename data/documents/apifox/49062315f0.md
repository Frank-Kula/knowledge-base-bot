# 发送请求

## Docs

- [Apifox 是否支持调试 Socket.IO？](https://docs.apifox.com/5837658m0.md): Apifox 支持调试 Socket.IO 的接口。请查看[帮助文档](//docs.apifox.com/6107994m0)。
- [Apifox 是否支持在 WebSocket 接口中使用前置/后置脚本或者断言？](https://docs.apifox.com/5837836m0.md): 目前还不支持，不过这个功能会在后续版本中考虑加入。
- [为什么发送请求的时候，出现"socket hang up"？](https://docs.apifox.com/5881479m0.md): "socket hang up" 错误表示连接中断，Apifox 没有收到任何响应，或者收到的响应不完整。这可能是由多种原因导致的：
- [为什么发送请求，提示“Invalid URI xxx"?](https://docs.apifox.com/6534734m0.md): 1. 检查该接口是否指定到前置 url 为空的服务
- [为什么同一个接口客户端请求成功，在web端调试会出现错误："无法请求地址" ？](https://docs.apifox.com/5885206m0.md): ### 在线文档或web端调试时出现无法请求地址？
- [接口响应过大时，为什么 Apifox 会报错提示超出 nodejs 字符串最大长度？](https://docs.apifox.com/5894955m0.md): Apifox 的前端在处理过大的接口响应时，可能会遇到与 Node.js 字符串长度限制相关的问题。Node.js 使用 V8 引擎，而 V8 引擎对单个字符串的长度有限制，通常在 337MB 左右。当响应大小超过这个限制时，可能会导致内存不足或字符串处理错误。
