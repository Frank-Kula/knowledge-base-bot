# 了解更多

现在你已经知道怎么使用 Apifox 来调试 API 了，接下来深入了解它的强大功能吧！

## 定义你的 API

- **[导入多种格式的 API 文档](https://docs.apifox.com/import-and-export.md)**  

   Apifox 允许你导入多种格式（如 OpenAPI/Swagger 和 Postman）的 API 文档到现有的 API 项目中，方便与现有工作流程无缝对接。
   
- **[智能解析 API](https://docs.apifox.com/extract-response-example.md)**

    Apifox 能够智能解析并自动生成 API 定义 *（<Tooltip tip="也称 OpenAPI/Swagger 规范，在 Apifox 中表示接口文档。">API specification</Tooltip>）* ，简化现有 API 文档化的过程。
    
- **[多分支协作开发](https://docs.apifox.com/sprint-branch.md)**

   团队可以在不同分支上同时处理 API 的不同版本，提高团队协作和版本控制的灵活性。

## 简化 API 调试

- [**自动生成请求参数与请求体**](https://docs.apifox.com/request-params-and-body.md)

   根据你定义的接口自动生成对应的请求参数和请求体，不需要手动输入，减少人为错误，确保发送的请求与接口定义完全一致。
   
- [**自动校验响应**](https://docs.apifox.com/validate-response.md)
   
   自动检查接口返回的响应，校验响应是否符合预先定义的数据结构，快速发现接口返回与预期不符的情况。
   
- [**兼容 Postman 脚本**](https://docs.apifox.com/scripts.md)

   完全支持并兼容 Postman 格式的脚本，可以直接导入现有的 Postman 脚本，让熟悉 Postman 的团队无缝切换。
   
- [**执行数据库操作**](https://docs.apifox.com/database.md)
   
   可以直接连接数据库，并在测试接口时执行数据库操作，支持数据库的增删改查功能。
    
- [**调用外部程序**](https://docs.apifox.com/call-external-programs.md)

   支持与多种编程语言集成，帮助团队根据需求扩展功能，自定义工作流程。
    
- [**兼容微服务场景**](https://docs.apifox.com/environments-and-services.md)

   适配微服务系统，帮助管理复杂的分布式 API，支持微服务间的切换测试。

    
- [**自动生成 Mock 数据**](https://docs.apifox.com/smart-mock.md)

    根据你定义的接口自动生成 Mock 数据，帮助前端团队独立开发，加速进度。
    
- **[代码生成](https://docs.apifox.com/generate-code.md)**

    根据 API 定义自动生成请求代码和业务逻辑代码。

## 发送不同类型的请求

- **[认证与授权](https://docs.apifox.com/authentication-and-authorization.md)**

    轻松设置身份验证和授权方法，安全地测试接口。

- **[GraphQL 调试](https://docs.apifox.com/graphql.md)**

    简单构建和发送复杂的 GraphQL 查询和变更请求。
    
- **[WebSocket 调试](https://docs.apifox.com/websocket.md)**

    使用 WebSocket 请求测试实时通信场景。
    
- **[SOAP/WebService](https://docs.apifox.com/soap.md)**

    支持 SOAP 和 WebService 请求，便于与旧协议对接。
    
- **[gRPC 调试](https://docs.apifox.com/grpc.md)**

    支持 gRPC 协议调试，轻松测试微服务接口调用。
    
- **[Dubbo 调试](https://docs.apifox.com/create-dubbo-api.md)**

    提供 Dubbo 协议支持，便捷调试 Java 分布式服务框架接口。

## 创建真实的 Mock 数据

- **[自动生成 Mock 数据](https://docs.apifox.com/smart-mock.md)**

    能自动生成符合接口结构的 Mock 数据。
    
- **[使用响应示例数据](https://docs.apifox.com/mock-priority-rules.md)**

    利用接口结构中预设的响应示例，确保 Mock 数据与真实接口保持一致。
    
- **[自定义响应数据](https://docs.apifox.com/custom-mock.md)**

    为特定的 Mock 场景定义自定义响应。
    
- **[动态 Mock 响应](https://docs.apifox.com/custom-mock.md)**

    根据输入参数生成变化的 Mock 响应。
    
- **[交互式 Mock 场景](https://docs.apifox.com/mock-scripts.md)**

    通过脚本动态修改请求参数，实现更灵活的接口测试。
    
- **[云端 Mock 服务](https://docs.apifox.com/cloud-mock.md)**

    设置云托管的 Mock 服务，提供可靠且随时可用的 Mock API 环境。
