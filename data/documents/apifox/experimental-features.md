# 实验性功能

> Apifox 版本需 ≥ 2.7.14

在使用 Apifox 进行接口调试和测试的过程中，稳定性与性能表现往往与本地设备资源密切相关。为了帮助用户在不同配置的设备上获得更流畅的使用体验，Apifox 提供了多项实验性功能。你可以根据设备性能和使用场景，灵活开启这些选项，以平衡资源占用与执行效率。

## 使用独立进程执行 API 请求

开启后，Apifox 在执行 API 请求时，会使用独立进程。这可以避免内存占用过高，减少卡顿情况，建议内存容量较小的电脑开启该选项。

## 使用独立进程运行测试场景

开启后，Apifox 在运行测试场景时，会使用独立进程。这可以让测试场景运行更快，但可能会消耗更多硬件性能。当你的硬件内存压力过高时，此设置不会生效。

## 优化本地服务进程内存

本地服务进程是本地 Mock 和导出 OpenAPI 文档的服务进程。开启后，该进程的内存将被自动回收。这可以降低内存占用，但可能会让本地服务的请求变慢。建议内存容量较小的电脑开启该选项。
