# 为什么在 WEB 端中修改 Referer 请求头无效？

#### 1. 为什么在 Apifox Web 端中修改 Referer 请求头无效？
- Referer属于浏览器的`禁止修改的请求头`，受浏览器安全策略限制，前端脚本（如网页代码、浏览器扩展）无法手动修改或覆盖该请求头。这是浏览器为防止跨站请求伪造（CSRF）等安全风险设置的保护机制。


#### 2. 那如何修改Referer请求头？
- 建议使用客户端进行尝试，客户端不受浏览器安全策略限制，支持自定义包括 Referer 在内的各类请求头，适合调试需要特定 Referer 的接口。

#### 3. 浏览器禁止修改的请求头还有哪些？
以下请求头属于浏览器禁止修改的范围（包含但不限于）：
- Accept-Charset
- Accept-Encoding
- Access-Control-Request-Headers
- Access-Control-Request-Method
- Connection
- Content-Length
- Cookie
- Date
- DNT
- Expect
- Host
- Keep-Alive
- Origin
- Permissions-Policy
- Proxy-*（所有以Proxy-为前缀的请求头）
- Sec-*（所有以Sec-为前缀的请求头）
- Referer
- TE
- Trailer
- Transfer-Encoding
- Upgrade
- Via
- X-HTTP-Method（仅当值包含禁止的方法名，如CONNECT、TRACE、TRACK时）
- X-HTTP-Method-Override（仅当值包含禁止的方法名时）
- X-Method-Override（仅当值包含禁止的方法名时）

可参考MDN官方文档：[https://developer.mozilla.org/en-US/docs/Glossary/Forbidden_request_header](https://developer.mozilla.org/en-US/docs/Glossary/Forbidden_request_header)，其中详细列出了浏览器禁止修改的请求头及其原因。
