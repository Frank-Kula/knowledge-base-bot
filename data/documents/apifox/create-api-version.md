# 创建 API 版本



进入某个项目后，在目录树上方的分支切换组件中，点击后可以看到“API 版本”选项。选择完成后即可看到当前项目包含的全部 API 版本。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/484929/image-preview)
</Background>


点击新建 API 版本，会需要输入版本的名称，以及选择版本的初始内容。默认从其它现有 API 版本中创建全量副本。如果选择了这个方式，新建的 API 版本中会包含选中版本中全部资源的副本。如果选择空白，则会有一个没有任何内容的新版本。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/484930/image-preview)
</Background>


点击保存后，创建新 API 版本成功，会自动打开新的 API 版本。选择了从现有版本创建副本，则会看到新的版本中有被选中 API 版本的全部资源副本。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/484931/image-preview)
</Background>


在新的 API 版本中点击任意一个资源，即可对资源像主分支内容一样进行编辑。这里修改的内容跟选择复制的 API 版本内容独立不会相互影响。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/484932/image-preview)
</Background>


通过以上方式，即可创建出一个 API 版本。并且根据实际情况的需要，对 API 版本中的具体资源进行修改调整，从而形成一个有独立内容的 API 版本。
