## 开始使用 Apifox

在使用 Apifox 开发 API 时，主要有两种核心开发模式。一种是 **“文档模式”**，另一种是 **“调试模式”**。

两种模式各有优势，你可以根据团队的开发习惯选择任一模式开始使用 Apifox。

<Background>

![CleanShot 2025-03-19 at 14.12.49@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/504770/image-preview)
</Background>


## 文档模式

在文档模式下，你需要在编写代码前先规划和定义 API 规范（也就是先写 API 文档）。这有助于确保所有团队成员在开发前对 API 的功能和数据结构有清晰的理解。

对于这种模式，推荐使用 Apifox 的[文档模式](https://docs.apifox.com/design-and-request-mode.md)。

<Background>
![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/505429/image-preview)
</Background>

使用文档模式的团队可以按照以下步骤快速上手：

<Steps>
<Step>
[新建接口](https://docs.apifox.com/create-api.md)
</Step>
<Step>
[发送接口请求](https://docs.apifox.com/send-api-request.md)
</Step>
<Step>
[添加断言](https://docs.apifox.com/add-assertions.md)
</Step>
<Step>
[新建测试场景](https://docs.apifox.com/create-test-scenario.md)
</Step>
<Step>
[分享 API 文档](https://docs.apifox.com/share-api-documentation.md)
</Step>
<Step>
[了解更多功能](https://docs.apifox.com/learn-more.md)
</Step>
</Steps>

## 调试模式

在调试模式下，团队可根据当前需求先编写代码来开发 API，并通过[快捷请求](https://docs.apifox.com/quick-request.md)来验证功能效果，以快速发现和解决问题，该模式特别适合需要高频迭代的项目。

对于这种情况，推荐使用 Apifox 的[调试模式](https://docs.apifox.com/design-and-request-mode.md)。

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/505431/image-preview)
</Background>

使用调试模式的团队可以按照以下步骤快速上手：


<Steps>
<Step>
[发送快捷请求并保存为接口](https://docs.apifox.com/quick-request.md)
</Step>
<Step>
[添加断言](https://docs.apifox.com/add-assertions.md)
</Step>
<Step>
[新建测试场景](https://docs.apifox.com/create-test-scenario.md)
</Step>
<Step>
[分享 API 文档](https://docs.apifox.com/share-api-documentation.md)
</Step>
<Step>
[了解更多功能](https://docs.apifox.com/learn-more.md)
</Step>
</Steps>
