# 如何将 Postman 的环境变量迁移到 Apifox？

在 Postman 中往往会使用一个变量 *（例如`{{baseUrl}}`）* 来指定前置 URL *（例如 `http://127.0.0.1:8080`）*，迁移到 Apifox 时，其前置 URL 会自动转换为 Apifox 「环境管理」中的「服务」。

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/490264/image-preview)

Postman 中的变量分为三种类型：集合变量、环境变量和全局变量。其中，**集合变量**会随项目的 JSON 文件一起导出。在导入到 Apifox 时，这些集合变量会自动迁移为 Apifox 的全局变量。


![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/490267/image-preview)



但是，Postman 中的**全局变量**和**环境变量**不会包含在导出的 JSON 文件中。因此，这些变量需要单独导出为 JSON 文件，然后再迁移到 Apifox。

要导出 Postman 的全局变量，依次选择 “Environments -> Globals -> Export”，然后导出为 JSON 文件。

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/490272/image-preview)

要导出 Postman 的环境变量，只需在 Environments 中，点击环境名称右侧的三个小点图标，然后选择「Export」。


![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/490273/image-preview)

全局/环境变量的 JSON 文件导出之后，就可以导入到 Apifox 中，在项目的左侧边栏依次选择 “项目设置 -> 导入数据 -> Postman”，然后上传已导出的全局/环境变量文件，确定导入即可。


![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/490274/image-preview)

导入完成后，你可在 Apifox 的环境管理中查看导入的变量。注意，导入的只是变量，「前置 URL」还需要手动在这里添加。


![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/490277/image-preview)

并且，如果设置有多个「前置 URL」，还需要在「接口」或者「目录」层级中确认是否是该接口所需的。

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/490278/image-preview)
