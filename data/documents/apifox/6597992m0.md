# 如何在 Apifox 中导入 Swagger/OpenAPI 时自动将接口分组？

在 Apifox 中导入 Swagger/OpenAPI 文件时，接口的目录会基于接口的 tags 定义来导入，系统会将接口的 tags 数组的第一个值设置为该接口的目录。


如果 tags 值中存在斜杠（“/”），那么 “/” 将用作分隔符来区分父目录和子目录结构。


例如，当接口标签设置为 ["Orders/User Orders/Sub Folder"] 时，导入后它会自动位于 Orders->User Orders->Sub Folder 这样的分层目录中。如果接口没有标签，则接口将导入目标目录的根目录中。

此外，在导入文件时，如果希望导入后的接口目录与导入源中的目录完全相同，需要开启 “更新接口所在目录” 开关。
