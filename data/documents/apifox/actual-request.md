# 实际请求

如果 API 请求没有按预期工作，可能有多个原因。你可以使用 Apifox 控制台来诊断并排查问题。以下是一些常见问题及其可能的解决方案：

## 实际请求

发送请求后，你可以在响应控制台中切换到 “实际请求” 标签，查看实际发送请求的 URL、请求头和请求体。

## 控制台调试

Apifox 控制台允许你随时输出日志，以帮助定位请求或脚本中的问题。支持以下日志语句：

- `console.log()`
- `console.info()`
- `console.warn()`
- `console.error()`
- `console.clear()`

变量提取、数据库操作等也会自动输出到控制台。

## 常见问题及解决方案

| **问题**               | **解决方案**                                                                                                                                  |
|------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------|
| **连接问题**           | 如果 Apifox 无法发送请求，可能存在连接问题。尝试在浏览器中打开网页，检查网络连接是否正常。                                              |
| **防火墙**             | 防火墙可能会阻止非浏览器的连接。联系网络管理员，确保 Apifox 可以正常访问外部网络。                                                      |
| **代理设置**           | 如果使用代理服务器发送请求，验证代理设置是否正确。Apifox 默认使用操作系统网络设置中的代理配置，控制台提供详细的调试信息。                 |
| **SSL 证书问题**       | 如果遇到 HTTPS 连接问题，可以通过设置图标禁用 SSL 证书验证。如果问题仍然存在，检查服务器是否需要客户端证书，并在设置中配置相应证书。 |
| **客户端证书缺失**     | 如果 API 服务器需要客户端证书，可以在设置中添加并配置证书。                                                                                |
| **错误的请求 URL**     | 检查 URL 中的路径参数或变量是否正确，确保最终请求地址无误。未解析的请求变量可能导致错误的服务器地址。                                      |
| **错误的协议**         | 确保 URL 中使用了正确的协议（http:// 或 https://）。                                                                                       |
| **超时设置过短**       | 如果 Apifox 的超时设置过短，可能会在请求完成之前导致超时错误。可以通过设置图标增加超时时间来避免此问题。                                   |
| **无效响应**           | 如果服务器返回的响应包含编码错误或无效头信息，Apifox 可能无法解析响应。                                                                  |
| **TLS 版本不兼容**     | Apifox 支持 TLS 1.2 及以上版本。如果服务器使用较旧的 TLS 版本，可能与 Apifox 不兼容。                                                      |
| **Apifox 错误**         | 如果怀疑 Apifox 向 API 服务器发送了错误请求，检查服务器日志并验证。如果问题无法解决，请通过 GitHub 问题追踪器联系 Apifox 支持团队。       |
| **未解析的变量**       | 未解析的变量在请求中未定义或超出了有效范围。检查变量定义和作用域。                                                                        |
| **CORS 问题**          | 如果 Apifox 网页应用遇到 CORS 错误，尝试使用 Apifox Agent 或 Apifox Client 以避免跨源资源共享问题。                                         |
