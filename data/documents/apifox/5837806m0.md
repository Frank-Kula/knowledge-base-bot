# 为什么参数值中的"+"会被解码成空格？


在 URL 编码规则中，表单数据里的空格字符需要编码成 "+"，因为在 URL 中直接使用空格是不安全的。

虽然`%20`也是空格的 URL 编码形式，但相对来说又长又不容易阅读。所以当 Web 服务器接收并解码 URL 或表单数据时，会把 "+" 解码成空格。这是因为在 application/x-www-form-urlencoded MIME 类型数据中，"+" 被用来代替空格。

### 如何在参数值中编码"+"：

如果你想在参数值中编码 "+"，可以用自动编码功能轻松搞定。具体这么做：


<Steps>
  <Step>
    选中文本：把含有空格的文本选中
  </Step>
  <Step>
    右键点击：点击鼠标右键
  </Step>
  <Step>
    选择自动编码：从右键菜单中选择编码选项
  </Step>
    <Step>
    查看编码结果：选择编码选项后，文本会自动转换
  </Step>
</Steps>
