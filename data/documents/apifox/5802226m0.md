# 接口签名如何处理


## 期望效果

**统一处理签名**：避免每个接口单独处理签名逻辑，提高代码复用性和维护性。


## 实现思路

1. **创建公共脚本**  
   - 利用[公共脚本](https://docs.apifox.com/common-scripts.md)实现签名逻辑。  
   - [读取接口请求参数](https://docs.apifox.com/access-modify-request-data.md)，使用[内置 JS 类库](https://docs.apifox.com/js-libraries.md)，如 [crypto-js](https://www.npmjs.com/package/crypto-js) 生成签名。

2. **注入签名参数**  
   - **方案一**：直接在脚本中修改请求信息，动态添加签名参数。  
   - **方案二**：将生成的签名写入环境变量，接口中通过引用环境变量设置签名参数。

3. **引用公共脚本**  
   - 在需要签名的接口的[前置脚本](https://docs.apifox.com/pre-request-scripts.md)中调用公共脚本。

4. **支持其他编程语言签名**  
   - 若签名算法由其他编程语言实现，可使用 `fox.execute` 方法[调用外部程序](https://docs.apifox.com/call-external-programs.md)，确保灵活适配。



## 示例：微信支付签名算法

以下示例基于[微信支付签名算法](https://pay.weixin.qq.com/wiki/doc/api/jsapi.php?chapter=4_3)。  

**规则**：  
1. 将所有非空参数按参数名的 ASCII 排序（字典序）。  
2. 按照 `key1=value1&key2=value2` 的形式拼接成字符串 `stringA`。  
3. 在 `stringA` 后拼接密钥，生成 `stringSignTemp`，然后进行 MD5 加密，并转为大写，得到最终签名。

**注意**：  
- 空值参数和 `sign` 参数本身不参与签名。  
- 参数名区分大小写。  

---

**公共脚本实现：**

```javascript
// 获取预先设置的 APPKEY
let key = pm.environment.get('APPKEY');

// 收集需要参与签名的参数
let param = {};

// 处理 Query 参数
pm.request.url.query.each(item => {
  if (!item.disabled && item.value !== '') {
    param[item.key] = item.value;
  }
});

// 处理 Body 参数
if (pm.request.body) {
  let bodyData;
  switch (pm.request.body.mode) {
    case 'formdata':
      bodyData = pm.request.body.formdata;
      break;
    case 'urlencoded':
      bodyData = pm.request.body.urlencoded;
      break;
    case 'raw': {
      const contentType = pm.request.headers.get('content-type');
      if (contentType && contentType.toLowerCase().includes('application/json')) {
        try {
          const jsonData = JSON.parse(pm.request.body.raw);
          for (let key in jsonData) {
            if (jsonData[key] !== '') {
              param[key] = jsonData[key];
            }
          }
        } catch (e) {
          console.log('请求 body 不是 JSON 格式');
        }
      }
      break;
    }
    default:
      break;
  }
  if (bodyData) {
    bodyData.each(item => {
      if (!item.disabled && item.value !== '') {
        param[item.key] = item.value;
      }
    });
  }
}

// 排序参数
const sortedKeys = Object.keys(param).filter(key => key !== 'sign').sort();
const paramString = sortedKeys.map(key => `${key}=${encodeURIComponent(param[key])}`).join('&');
const stringSignTemp = `${paramString}&key=${key}`;

// 生成签名
const sign = CryptoJS.MD5(stringSignTemp).toString().toUpperCase();

// 注入签名到请求中
pm.request.url.query.upsert({ key: 'sign', value: sign });
```

## 示例：百度翻译签名

**规则**：  
1. 按顺序拼接 `appid + query + salt + 密钥`，生成 `字符串1`。  
2. 对 `字符串1` 进行 MD5 加密，生成 32 位小写签名。

**公共脚本实现：**

```javascript
// 获取环境变量
const appid = pm.environment.get('APPID');
const secretKey = pm.environment.get('SECRET_KEY');

// 获取 Query 参数
const queryParams = pm.request.url.query;
const q = queryParams.get('q');

// 随机生成 salt
const salt = Math.floor(Math.random() * (65536 - 32768 + 1)) + 32768;

// 拼接字符串
const signString = appid + q + salt + secretKey;

// 生成签名
const sign = CryptoJS.MD5(signString).toString();

// 注入参数到请求中
queryParams.upsert({ key: 'salt', value: salt });
queryParams.upsert({ key: 'sign', value: sign });
```
