# Jenkins 定时触发任务


文中 Jenkins 版本为 `2.414`，不同版本界面可能会有出入，但操作逻辑差不多是一致的。

## 配置 NodeJS 环境

运行 Apifox CLI 前需确保 Node.js 版本号 `>= v14.20.1`，因此需要先在 Jenkins 环境中配置 NodeJS 依赖。

1、打开 Jenkins 插件管理，找到 NodeJS 插件，安装 NodeJS 并重启 Jenkins。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/488345/image-preview)
</Background>

2、NodeJS 安装完毕，需要在全局配置工具里新建 NodeJS，配置版本号（需 >= v10）和包名 `apifox-cli`。

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/488346/image-preview)
</Background>


## 运行 Apifox CLI 命令

**1、创建新任务**

创建一个新任务并选择任务类型为 `Freestyle project`。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/488348/image-preview)
</Background>


**2、设置定时触发**

在 「项目配置（Configure）-> 构建触发器」处，勾选「定时构建（Build periodically）」，然后在日程表里写上 cron 表达式即可，比如要每分钟触发一次，你可以使用 cron 表达式：`* * * * *`。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/488349/image-preview)
</Background>


**3、设置构建环境**

在 「构建环境（Build Environment）」 中设置 Node 信息，如果配置有多个 Node，可以根据需要选择对应版本。

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/488350/image-preview)
</Background>


**4、添加构建步骤**

在 Windows 系统下，你可以选择 “Execute Windows batch command”，Mac 系统下，你可以选择 “Execute shell”。然后将 Apifox Cli 的命令拷贝进去，保存并运行即可。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/488351/image-preview)
</Background>
