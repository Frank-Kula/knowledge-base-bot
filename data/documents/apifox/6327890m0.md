# 通过 MCP 使用公开发布的 API 文档

Apifox MCP Server 可以将公开发布的在线文档提供给 AI 使用。

仅支持公开发布、任何人都可以直接访问的在线文档，**不支持设置了密码、邮箱白名单、IP 白名单的在线文档**。如果不希望公开接口文档，仅仅是内部人员可以访问，建议使用项目 ID 和个人访问令牌读取 Apifox 项目数据，详见：[通过 MCP 使用项目内文档](https://docs.apifox.com/6327888m0.md)。

![f08ea56d-7779-414d-b8b9-c8aedc219e82.gif](https://api.apifox.com/api/v1/projects/5097254/resources/507182/image-preview)



## 为在线文档开启 MCP 功能

>  Apifox 版本 `≥ 2.7.2`

<Steps>
  <Step title="开启 MCP 服务">
    进入 Apifox 项目，依次点击「分享文档 -> 发布文档站 -> AI 功能」，开启 MCP 服务。
      
<Background>

![CleanShot 2025-03-26 at 13.40.27@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/507152/image-preview)
</Background>

  </Step>
  <Step title="获取配置文件">
    开启后，访问在线文档的接口时，页面将显示 「MCP」按钮。
      
:::tip[]
按钮会显示在接口页面中，而不是 Markdown 页面。如果未显示，请检查你发布的在线文档是否启用了密码保护、IP 白名单等访问限制，在这些受限访问的情况下，按钮将无法显示。
:::
      
<Background>

![CleanShot 2025-07-25 at 11.18.47@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/549248/image-preview)
</Background>
点击后，将弹出详细的使用说明和 MCP 配置文件，文件内已自动填充当前文档的 `site-id`。只需复制该配置，便可用于 IDE 中接入 MCP 服务。
      
<Background>
![CleanShot 2025-03-26 at 13.43.11@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/507156/image-preview)
</Background>
  </Step>
</Steps>


## 配置 MCP 客户端

### 前置条件

- 已安装 Node.js 环境（版本号 >= 18，推荐最新的 LTS 版本）
- 任意一个支持 MCP 的 IDE：
  - Cursor
  - VSCode + Cline 插件
  - 其它
- 已从 Apifox 在线文档复制了 MCP 的 JSON 配置


### 在 Cursor 中配置 MCP


<Steps>
  <Step title="编辑 MCP 配置文件">
    打开 Cursor 编辑器，点击右上角 「设置」图标，选择左侧「MCP」选项，点击「+ Add new global MCP server」按钮。
      
<Background>

![CleanShot 2025-03-26 at 12.07.53@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/507116/image-preview)
</Background>

  </Step>
  <Step title="添加配置内容">
在打开的 `mcp.json` 文件中，添加从 “在线文档” 中复制的的 MCP 配置，例如：
      
    <Tabs>
      <Tab title="macOS / Linux">
        ```json
        {
          "mcpServers": {
            "API 文档": {
              "command": "npx",
              "args": [
                "-y",
                "apifox-mcp-server@latest",
                "--site-id=123456"
              ]
            }
          }
        }
        ```
      </Tab>
      <Tab title="Windows">
        ```json
        {
          "mcpServers": {
            "API 文档": {
              "command": "cmd",
              "args": [
                "/c",
                "npx",
                "-y",
                "apifox-mcp-server@latest",
                "--site-id=123456"
              ]
            }
          }
        }
        ```
      </Tab>
    </Tabs>
      
  </Step>
  <Step title="验证配置">
    配置完成后，你可以通过向 AI 询问以验证连接是否正常工作（Agent 模式），例如：

```
请通过 MCP 获取 API 文档，并告诉我项目中有几个接口
```
      
如果 AI 能够返回你 Apifox 项目中的 API 信息，说明连接成功。
<Background>

![CleanShot 2025-03-26 at 12.13.11@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/507120/image-preview)
</Background>
  </Step>
      


</Steps>

## 注意事项

- 如果需要使用到多个 API 文档，在配置文件里添加配置多个 MCP Server 即可（不同文档有不同的`<site-id>`）。
- 使用私有化部署版本的用户，请在 IDE 的 MCP 的配置文件添加参数："--apifox-api-base-url=`<私有化部署服务器的 API 地址，以 http:// 或 https:// 开头>`"。另外，请确保网络可以正常访问 `www.npm.com`。
      
    ```json
     {
      "mcpServers": {
        "API 文档": {
          "command": "npx",
          "args": [
            "-y",
            "apifox-mcp-server@latest",
            "--site-id=123456",
            // 私有化部署必须添加以下参数
            "--apifox-api-base-url=<私有化部署服务器的API地址>"
          ]
        }
      }
    }
    ```

## 常见问题


<Accordion title="Windows 系统配置问题" defaultOpen>

如果在 Windows 操作系统上，前述配置无法正常工作，请使用以下配置：

```json
{
  "mcpServers": {
    "API 文档": {
      "command": "cmd",
      "args": [
        "/c",
        "npx",
        "-y",
        "apifox-mcp-server@latest",
        "--site-id=123456"  // 替换为你的 Site ID
      ]
    }
  }
}
```
</Accordion>



<Accordion title="Node.js 版本问题" defaultOpen={false}>

如果遇到错误提示 Node.js 版本过低，请确保你安装了 18 或更高版本的 Node.js。你可以通过以下命令检查当前版本：

```bash
node -v
```
</Accordion>


<Accordion title="API 文档更新后，如何让 AI 读取最新数据？" defaultOpen={false}>
API 文档数据会缓存在本地，如果接口文档有更新，记得告诉 AI 刷新接口文档数据，以确保代码生成时使用的是最新版本。

例如：

```
请重新读取 API 文档数据，在 Pet DTO 里添加 API 文档新增的几个字段
```
</Accordion>
