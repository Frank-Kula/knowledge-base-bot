# 发送请求失败 “Error: getaddrinfo ENOTFOUND xxx.xxx.xxx.xxx” 的原因及排查方法

Apifox 中遇到 `Error: getaddrinfo ENOTFOUND xxx.xxx.xxx.xxx` 错误，通常表示 Apifox 无法解析请求中的域名或主机名，或者网络不可达。以下是可能的原因及排查方法：

### **1. URL 或 IP 地址错误:**
仔细检查请求的 URL 或 IP 地址是否拼写正确，包括协议 (http:// 或 https://)。尝试在浏览器中直接访问该 URL，看是否能正常打开。

### **2. DNS 解析失败:**
- 使用命令行工具测试解析：
```js
nslookup <域名>    # Windows/Linux/macOS
dig <域名>         # Linux/macOS
```
检查返回的 IP 地址是否正确。如果解析失败，可以尝试更换 DNS 服务器 (例如 8.8.8.8 或 114.114.114.114)。
- 检查本地 hosts 文件 (路径：`C:\Windows\System32\drivers\etc\hosts` 或 `/etc/hosts`)，确保没有错误或被篡改的域名解析条目。

### **3. 网络连接问题:**

- 使用 `ping xxx.xxx.xxx.xxx` 命令测试与目标服务器的连通性。如果无法 ping 通，可能是网络不通或防火墙拦截 (注意：某些服务器可能禁用了 ICMP)。
- 使用 `telnet xxx.xxx.xxx.xxx <端口号>` 命令测试目标服务器的端口是否开放 (例如 `telnet xxx.xxx.xxx.xxx 80` 测试 HTTP 的 80 端口，`telnet xxx.xxx.xxx.xxx 443` 测试 HTTPS 的 443 端口)。

### **4. 本地网络环境问题:**

- 尝试关闭 VPN 或代理，看看是否能解决问题。
- 检查防火墙或安全软件，确保没有阻止 Apifox 或相关端口的访问。
- 如果使用代理，请检查 Apifox 的代理设置是否正确 (设置路径：设置 -> 代理)。
- 目标服务器问题: 确认目标服务器是否正常运行，可以尝试通过其他设备或在线工具访问。检查域名解析是否指向正确的 IP 地址。

### **5. Apifox 配置问题:**

- 确保 URL 中的协议 (http/https) 和端口 (例如 :8080) 与服务器配置一致。
- 检查是否在 Apifox 中手动设置了错误的 Host 请求头。
- 尝试更新 Apifox 到最新版本，以修复可能的兼容性问题。
- 更改 [网络代理](https://docs.apifox.com/network-proxy.md) 配置


### **6. 本地环境问题:**

尝试重启路由器和计算机，或者更换网络环境 (例如从 Wi-Fi 切换到移动热点)。
