# 在线文档中调试 http 接口时报错"Mixed Content request rejected"？

### 在线文档页面调试 HTTP 接口会提示"Mixed Content request rejected"？
Apifox 文档页面是通过 `https://` 部署的，当在该页面调试 HTTP 接口时，若不通过代理、直接由浏览器原生发起请求，则会形成“HTTPS 页面请求 HTTP 接口”的混合内容场景。浏览器为保障安全，会认为 HTTPS 页面中加载的 HTTP 资源可能被篡改，进而主动拦截请求，导致调试失败。

### 为什么相同接口在客户端能正常调用，在文档页面却不行？
因为客户端不受浏览器的混合内容策略限制，无论接口是 HTTP 还是 HTTPS，只要网络连接正常，就能正常请求。而浏览器会执行严格的安全策略，所以会出现上述差异。

### 遇到此问题如何解决？
若在线文档用户遇到该问题，需要修改在线分享设置中的跨域代理选项。注意不要设置为`自动模式`或`不使用代理模式`。你可以根据实际情况选择`云端 Agent`,`浏览器扩展`或`自托管请求代理 Agent`模式。
