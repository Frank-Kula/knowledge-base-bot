# 基本概念

Apifox 提供了全面的管理功能，帮助你高效地组织、管理和协作 API 接口的设计、开发及文档编写。通过 **“团队”**、**“项目”** 和 **“组织”** 这三个层级的功能，你可以更加精细化地管理和控制不同范围内的 API 开发工作。


## 团队

在 Apifox 中，“团队” 是协作的核心单元。每个团队可以包含多个项目和成员，并且团队之间的数据和项目是互相隔离的。团队功能支持成员管理、权限控制和高效协作，你可以邀请成员加入团队，并为他们分配不同的角色和权限，确保项目的安全性。

团队成员可以共享资源、协作编辑和讨论，提高工作效率和项目质量。通过设置合理的权限，团队可以在不泄露敏感数据的前提下进行高效合作。

你可以阅读以下文档快速了解团队的功能和使用方法：

- [管理团队](https://docs.apifox.com/team-basic-operations.md)
- [管理团队成员](https://docs.apifox.com/team-member-management.md)

## 项目

“项目” 是管理 API 及相关文档的基本单位。每个项目代表一个独立的工作空间，你可以在项目中创建、编辑和测试 API，设计和管理 API 文档，并与团队成员协作完成任务。

通过创建多个项目，你可以根据需求对 API 和文档进行分类，进一步提升工作效率和可维护性。每个项目都有独立的设置和权限控制，确保项目安全和数据隐私。

## 组织

“组织” 是管理和协作的最高层级，通常对应一个公司或一个大的团队。组织能够集中管理多个团队，并对它们进行统一的权限配置和资源分配。通过组织功能，你可以跨团队地管理所有 API 项目，确保公司的数据安全、团队协作顺畅。

组织的设置允许管理员为不同的团队分配不同的权限，并对组织中的所有项目进行集中管理。每个组织都可以拥有多个团队，团队之间的数据和项目互相隔离，确保信息的安全性和隐私保护。

## 团队、项目和组织的区别

在 Apifox 中，**团队**、**项目**和**组织**是不同层级的概念，分别对应不同的管理和协作范围：

- **组织：** 是最高级别的管理单元，代表一个公司或大团队，负责集中管理多个团队。组织拥有对多个团队的管理权限。

- **团队：** 是组织中的子单元，通常对应一个部门或小团队，负责集中管理多个项目。团队拥有对多个项目的管理权限。

- **项目：** 是团队内的工作单元，用于管理特定的 API 资源和文档。每个项目是独立的工作空间，便于集中管理和协作。

简而言之：

- 组织用于管理多个团队。
- 团队负责管理具体的 API 项目。
- 项目则是管理和协作的具体单元，负责 API 的设计和开发。



## 组织 vs. 团队 —— 该怎么选？

在用 Apifox 的时候，很多人会纠结：到底是直接用组织来管，还是主要依靠团队？其实核心就看两个点——人数多少、业务范围有多大。

- **如果团队不大，业务也比较集中**

    比如你们就十来个人，做的也是同一个产品线，这种情况只用 团队 就够了。团队里可以管理项目、API 和成员，简单高效，不需要再加组织层。

- **如果规模很大，业务又分散**

    如果你们公司人多，业务线也五花八门，那最好先建一个 组织，再在组织里按业务划分多个 团队。每个团队只管自己的项目和成员，互不干扰，边界更清楚。
