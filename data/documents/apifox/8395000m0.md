# 新版 MCP 内测

📢 重磅内测 | Apifox 新 MCP 现已开启，邀您抢先体验！

🔥 核心升级亮点速览

**1. 协议全面升级**
 - 从 stdio 迁移为 Streamable HTTP，不再强依赖 Node 环境。
 
**2. 数据获取更全面、智能**
 - 从全量拉取缓存到本地，转变为按量按需精准拉取，减少占用。
 - 支持多项目、多分支、多模块、分文件夹。通过自然语言描述即可获取数据。

**3. 工具升级**
 - 工具数量从旧版的 3 项扩展至 18 项工具。涵盖接口、数据模型、Markdown 文档、测试用例的查、改、增、删等精细操作。
 - 开放接口 Beta 工具包。内置 100+ Apifox 开放接口，扩展除基础工具外的能力。
 - AI Agent 更友好的文件读取体验。MCP Servers 返回 AgentHits，指引 AI 将结构数据保存至本地。当 AI 操作不佳时，可以指引 AI 读取 `.apifox/` 下结构索引，也可以人工调整项目结构数据。 

**🚀 如何立即体验？**
1. 将以下配置复制到您的 Agent （Cursor、Claude Code）MCP 配置，并替换 `<access_token>` ：

- Cursor、Trae 等适用于 
```json
{
  "mcpServers": {
    "apifox-new-mcp": {
      "url": "https://apifox.com/api/v1/mcp",
      "headers": {
        "Authorization": "Bearer <access_token>",
        "X-Apifox-Api-Version": "2025-09-01"
      }
    }
  }
}
```
- Claude Code 适用于
```json
"mcpServers": {
    "apifox-new-mcp": {
      "headers": {
        "Authorization": "Bearer <access_token>",
        "X-Apifox-Api-Version": "2025-09-01"
      },
      "type": "http",
      "url": "https://api.apifox.com/mcp"
    }
}
```
- OpenCode 适用于
```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "apifox-new-mcp": {
      "headers": {
        "Authorization": "Bearer <access_token>",
        "X-Apifox-Api-Version": "2025-09-01"
      },
      "type": "remote",
      "enabled": true,
      "url": "https://api.apifox.com/mcp"
    }
  }
}
```
2. 获取您的 `<access_token>` ，在 Apifox  「设置」>「API 访问令牌」 页面新建一个令牌即可获取，替换上述占位。
3. AI 读取测试：「Apifox MCP 有哪些工具？」

- “列出 AI 相关接口的所有测试用例，评估一下当前的覆盖率”。
