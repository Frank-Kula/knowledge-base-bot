# 通知事件



通过创建通知事件，来指定具体某些操作会触发给通知对象发送通知。通知事件中包含事件名称、触发事件、通知对象。

## 触发事件

支持对以下操作触发通知：

| 触发事件         | 操作                                                                             |
| ---------------- | -------------------------------------------------------------------------------- |
| **接口变更**     | 新建接口<br> 修改接口<br> 删除接口                                               |
| **数据模型变更** | 新建数据模型<br> 修改数据模型<br> 删除数据模型                                   |
| **文档变更**     | 新建文档<br> 修改文档<br> 删除文档                                               |
| **导入数据**     | 手动导入<br> 自动导入                                                            |
| **响应组件变更** | 新建响应组件<br> 修改响应组件<br> 删除响应组件                                   |
| **迭代分支变更** | 合并迭代分支<br> 新建迭代分支<br> 归档迭代分支<br> 删除迭代分支<br> 复制迭代分支 |
| **API 版本变更** | 新建 API 版本<br> 删除 API 版本                                                  |

:::tip[]
注意，关于自动化测试的相关通知（自动化测试运行完成、持续集成运行完成、定时任务运行完成），需要在每个测试场景中进行单独的设置。这样的方式更灵活、更符合实际场景。
:::

## 通知对象

当触发事件被触发后，通知到的具体对象设置。直接选择项目中已经设置好的通知对象，可以同时选择多个通知对象。目前此处的通知对象，不支持选择通知渠道为邮件的通知对象。仅 自动化测试运行完成、持续集成运行完成、定时任务运行完成 这三种类型的事件支持使用通知渠道为邮件的通知对象。
