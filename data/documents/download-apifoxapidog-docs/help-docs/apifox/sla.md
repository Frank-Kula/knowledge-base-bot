# 服务等级协议


> 版本号：V1.0.0 发布日期：2024 年 06 月 01 日

本服务等级协议（Service Level Agreement，以下简称“SLA”）规定了 **Apifox** 网站 **https://apifox.com** 向**付费版客户**提供的产品服务可用性等级指标及赔偿方案。

## 第一条 定义

1.1 服务范围：客户通过 PC 客户端、PC 浏览器使用 Apifox 的服务。通过 API 等方式链接和访问 Apifox 的可用性，不在本协议所覆盖的范围内。

1.2 服务周期：一个服务周期为一个自然月，如果客户使用 Apifox 产品不满一个月则以一个月作为一个服务周期。

1.3 服务周期总分钟数：服务周期内的总天数 x 24（小时）x 60（分钟）计算。

1.4 服务不可用：当某一分钟内，客户组织内所有 Apifox 账户均不能登录 Apifox 服务，且访问 **https://app.apifox.com/** 域名下的所有链接，均返回错误代码 **5XX** 或超时。

1.5 服务不可用分钟数：在一个服务周期内，服务不可用的分钟数之和，即服务不可用分钟数。

1.6 月度服务费用：客户在一个自然月中就单个组织所支付的服务费用，如果客户购买了多个组织，每个组织单独计算。如果客户一次性支付了多个月份的服务费用，将按照所购买的月数分摊计算月度服务费用。

## 第二条 可用性

### 2.1 服务可用性

### 2.1.1 服务可用性计算公式

服务可用性以单个组织为维度，按照如下方式计算：

服务可用性 = (服务周期总分钟数 - 服务不可用分钟数) / 服务周期总分钟数 × 100%。

### 2.1.2 服务可用性承诺

Apifox 服务可用性不低于 99.8%，如未达到前述可用性承诺，客户可以根据本协议第 3 条约定获得赔偿。

### 2.2 数据存储持久性

### 2.2.1 数据存储持久性计算方式

数据存储持久性 =（每月用户存储的数据或文件总数 - 每月丢失的数据或文件数）/ 每月用户存储的数据或文件总数 × 100%

### 2.2.2 数据存储持久性承诺

Apifox 一个服务周期内数据持久性达 99.99999% 及以上，如未达到前述可用性承诺，客户可以根据本协议第 3 条约定获得赔偿。

### 2.3 除外情形

**因下述原因导致的服务不可用的时长不计入服务不可用时间：**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.1 Apifox 预先通知客户后进行系统维护所引起的，包括割接、维修、升级和故障演练；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.2 Apifox 所属任何设备以外的网络、设备故障或配置调整引起的；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.3 客户未遵循产品使用文档或使用建议引起的；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.4 客户的疏忽或由客户授权的操作所引起的；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.5 客户维护不当或保密不当致使数据、口令、密码等丢失或泄漏所引起的；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.6 客户本地环境不兼容引起的；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.7 客户的浏览器等应用程序或数据信息受到黑客攻击而引起的；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.8 不可抗力引起的；**

&nbsp;&nbsp;&nbsp;&nbsp; **2.3.9 客户自行升级操作系统所引起的；**
  
&nbsp;&nbsp;&nbsp;&nbsp; **2.3.10 客户的应用程序或安装活动所引起的。**

## 第三条 赔偿方案

### 3.1 赔偿标准

每个组织单独计算服务可用性，**按照下表中的标准计算赔偿金额，赔偿总额不超过未达到服务可用性当月客户就该组织支付的月度服务费用的 30%**。

| 可用性                            | 赔偿金额                |
|-----------------------------------|-------------------------|
| 服务可用性低于 99.8% 但等于或高于 95.00%  | 月度服务费用的 10%  |
| 服务可用性低于 95.00% 但等于或高于 90.00%  | 月度服务费用的 20%  |
| 服务可用性低于 90.00%             | 月度服务费用的 30%       |
| 数据存储持久性低于 99.99999%      | 月度服务费用的 30%       |

### 3.2 赔偿方式

Apifox 对于未达到服务可用性时，**赔偿方式仅限于用于购买该组织 Apifox 产品的代金券，** 且此代金券不可转让、不可兑现。

### 3.3 赔偿申请方式

**客户需通过 Apifox 官网公布的客服渠道，主动联系 Apifox 申请赔偿。Apifox 接到申请后，将在 10 个工作日内作出答复，并在两个自然月内，按照 3.1 条款中的约定，退回相应款项。**

### 3.4 赔偿申请时限

**客户可以在每个自然月第五个工作日后对上个月没有达到服务可用性承诺的组织提出赔偿申请。赔偿申请限于在 Apifox 服务没有达到服务可用性的相关月份结束后一个月内提出。超出申请时限的赔偿申请将不被受理。**

## 第四条 其他

**Apifox 有权不时地对本 SLA 条款作出修改，且毋须另行通知。修订后的协议一旦在网页上公布即有效代替原来的服务协议。您继续访问和使用 Apifox 即视为您接受修订后的服务协议。如您不同意 Apifox 对 SLA 所做的修改，您有权停止使用 Apifox 服务，如您继续使用 Apifox 服务，则视为您接受修改后的 SLA。**
