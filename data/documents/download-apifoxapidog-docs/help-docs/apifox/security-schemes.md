# 鉴权组件

## 什么是鉴权组件

鉴权组件是基于 OpenAPI 规范（OAS）设计的一种模板化认证方式，用于定义接口的认证需求。与直接在接口上配置认证方式不同，鉴权组件实现了认证配置的模板与值的分离，使团队可以更加灵活地管理和复用认证配置。

## 鉴权组件的优势

- **一处定义，多处使用：** 创建一次鉴权组件，可以在多个接口或目录中复用
- **支持组合**：一个接口可以使用多个鉴权组件
- **模板与值分离：** 定义鉴权模板与实际认证值分离，便于管理
- **避免鉴权信息泄露**：使用在线文档调试使用了鉴权组件的接口时，必须手动填写鉴权信息
- **层级继承：** 支持目录层级继承鉴权组件，减少重复配置
- **支持多种认证方式：** 包括 API Key、Bearer Token、Basic Auth、OAuth 2.0 等
- **符合 OpenAPI 标准：** 完全兼容 OpenAPI 3.0 规范的 Security Schemes


## 适用场景

- 项目中多个接口使用相同的认证方式
- 项目中一个接口使用多种认证方式
- 需要在不同环境中使用不同的认证值
- 团队协作开发 API，需要统一认证管理
- 生成标准 OpenAPI 文档时需要包含安全定义
