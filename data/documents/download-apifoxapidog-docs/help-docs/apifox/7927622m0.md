# 定时任务

测试套件支持设置定时任务，实现无人值守的自动化巡检。

## 新建定时任务

进入测试套件的「定时任务」标签页，点击「新建任务」。
- **运行周期：** 设置该定时任务以什么样的定时周期来进行执行，例如每周日晚上 11 点，或每间隔 6 小时执行一次。
- **运行于：** 实际消耗硬件资源运行测试套件的机器。所有测试套件中发起的请求，会从这里指定的机器发起，所以有可能因为请求机器的网络环境不同导致产生不同的测试结果。
- **通知设置：** 当任务运行失败（或完成）时，通过第三方集成（如飞书、钉钉、企业微信、Email）发送通知。

## 定时任务列表
在自动化测试的目录树中，保留了一个定时任务的入口，点击后可以用列表格式，看到当前项目内，全部套件中创建的定时任务，并进行开启/关闭的管理。

## 定时任务运行

定时任务根据运行周期设置，到时触发运行完成后，会将运行结果从 Runner 上传至服务端中，可以在客户端的 “定时任务 - 运行历史” 中选择查看定时任务的所有运行结果详情。



:::tip[]
建议定时任务开启通知，这样即可在运行完成后第一时间收到通知消息，从而及时查看任务运行结果。
:::
