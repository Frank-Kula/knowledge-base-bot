# 解决 ENOTFOUND: Couldn't resolve host 错误

`ENOTFOUND: Couldn't resolve host` 错误表示系统无法将你请求的域名或主机名转换为对应的 IP 地址。这是网络请求的第一个关键步骤——**域名解析失败了**。


## 错误原因

这个错误主要和 **DNS（域名解析系统）** 有关，而不是服务器是否开启的问题。

### 1. 域名拼写或格式错误
* **域名拼写错误：** 请求 URL 中的域名拼写有误，比如缺少`/`，或者多了一个`/`，导致 DNS 找不到对应的记录。
* **URL 格式问题：** 缺少 `http://` 或 `https://`。

### 2. 本地 DNS 或 Hosts 文件问题
* **DNS 缓存污染：** 你的电脑或路由器缓存了错误的 DNS 解析记录。
* **Hosts 文件干扰：** 你的本地 `hosts` 文件（用于本地覆盖 DNS 解析）中包含了错误的或过时的域名解析条目。

### 3. 网络环境或防火墙限制
* **网络连接中断：** 你的设备没有连接到互联网，或网络不稳定。
* **DNS 服务器故障：** 你的网络配置使用的 DNS 服务器（如路由器或 ISP 的 DNS）出现故障或无法访问。
* **VPN/代理干扰：** 正在使用的 **VPN 或代理** 服务配置错误，干扰了正常的 DNS 解析。
* **防火墙限制：** 你的本地防火墙或安全软件阻止了 DNS 查询请求（通常是 UDP 53 端口）。



## 解决方法

你可以按照以下步骤逐一排查和解决 `ENOTFOUND: Couldn't resolve host` 错误：

### 1. 安装插件或使用桌面版

如果你使用的是 Web 版，请安装 [浏览器扩展](https://docs.apifox.com/chrome-extension) 后再发送请求，也可以直接使用 [桌面版](https://docs.apifox.com/download)（推荐）。

### 2. 检查和修正域名/URL

* **核对域名：** 仔细检查你在 Apifox 中使用的完整 **URL** 或 **前置 URL**（Base URL）中的域名拼写是否完全正确，是否缺少了`/`，或者多了一个`/`。
    * 例如，确保你输入的是 `api.example.com` 而不是 `api.exampel.com`。
* **检查 URL 格式：** 确保 URL 包含正确的协议（`http://` 或 `https://`）。
* 若接口的前置 URL 含有环境变量，需要检查变量的值是否错误。

### 3. 刷新和检查本地解析

* **浏览器测试：** 尝试在你的**浏览器**中直接访问这个 URL。如果浏览器也无法访问，说明问题出在你的电脑或网络上。

* **刷新 DNS 缓存：**
    * **Windows:** 打开命令提示符（CMD），输入 `ipconfig /flushdns` 并回车。
    * **macOS:** 打开终端，输入 `sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder` 并回车。
   
* **检查 Hosts 文件：**
    * 检查你的本地 **`hosts` 文件**
      Windows 路径：`C:\Windows\System32\drivers\etc\hosts`

      macOS / Linux 路径： `/etc/hosts`
    * **删除或注释掉**（在行首加 `#`）任何与你请求的域名相关的解析记录，然后保存文件。

### 4. 检查网络和代理配置

* **更换 DNS：** 暂时将你的网络 DNS 设置更改为公共 DNS（例如 Google DNS: 8.8.8.8 或 Ali DNS: 223.5.5.5），然后再次尝试请求。
* **检查 VPN/代理：** 如果你正在使用 VPN 或代理，请：
    * 尝试**关闭 VPN/代理** 后发送请求。
    * 如果需要使用代理，请进入 Apifox 的 **“设置 -> 网络代理”** 页面，确保代理配置信息是完全正确的。

    </Frame>
* **重启网络设备：** 尝试重启你的路由器和调制解调器，清除潜在的网络问题。
