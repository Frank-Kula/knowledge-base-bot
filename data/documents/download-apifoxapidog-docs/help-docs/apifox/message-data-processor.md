# 报文数据处理器


数据处理器主要用来对发送或接收到的报文进行转换处理，包括以下 2 种数据处理器：

1. 请求报文处理器
2. 返回报文处理器

系统内置了一些常用的数据处理器，用户也可以添加自定义函数（例如 `Javascript` 代码），以满足各种场景的需要。

## 1. 请求报文处理器

用途：运行接口时对输入数据进行自动封装后再发送。

使用场景示例：

1. 发送报文时，自动计算包体内容长度，添加到报文头部，然后再发送。
2. 输入的 JSON 数据，转换为 XML 格式后在发送。

系统内置了一些常用的 “请求报文处理” 方法，如有特殊场景，用户可以添加 “自定义请求报文处理器”。

## 2. 返回报文处理器

用途：对返回的报文数据封装后再展示。

使用场景示例：

1. 展示时去除报文头部。
2. 将返回的 XML 转换成 JSON 格式。

系统内置了一些常用的 “返回报文处理”，如有特殊场景，用户可以添加 “自定义返回报文处理器”。

## 3. 添加自定义处理器

点击 “项目设置 -> 自定义函数（Socket）” 添加自定义处理器。
