# 为什么 curl 请求正常返回 200，但在 Apifox 中请求却返回 400？

如果 curl 请求正常，但在 Apifox 中相同的请求返回 400 错误，很可能是 Apifox 中的请求头设置与 curl 不同导致的。请检查以下几点：

- 请求头： 切换到 Apifox 的“Header”标签页，确认是否关了几个系统自动生成的 Header。Host头缺失通常会导致 400 错误。

- 大小写： 仔细对比 curl 命令中的请求头和 Apifox 中的请求头，确保两者完全一致。有些服务器对请求头大小写敏感。
- 请求体： 确认请求体的内容和格式与 curl 命令中的完全一致。
- 认证信息： 如果接口需要认证，请确保 Apifox 中正确配置了认证信息，例如 API Key、Bearer Token 等。
