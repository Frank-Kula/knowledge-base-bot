# 为什么测试场景 Apifox 客户端运行没问题，但在 Apifox CLI 或 Runner 中运行时却出错了？



最常见的问题涉及**变量**的使用。

在 Apifox 本地客户端运行时，使用的是存储在你客户端中的变量的 “本地值”，这个值不会同步到服务器。但在 CLI 或 Runner 中运行时，默认使用的是存储在服务器上的 “远程值”。

:::highlight green 📌
方案一：
为了确保在本地和远程都能顺利运行，你需要正确设置[本地值和远程值<Icon icon="material-outline-north_east"/>](/全局环境临时变量-5537409m0#远程值和本地值)，并保持它们一致。
:::

:::highlight green 💡
方案二：
CLI 命令或者 Runner 配置中直接设置使用本地值。
#### CLI 设置
- 切换到 cli 新版命令，“环境/全局变量值”切换成“使用测试场景中的本地值”
- 下载下图所示的变量文件到 cli/jenkins 的运行路径下，使得 cli 命令可以找到文件
- 运行引用了文件的新 cli 命令（ cli 版本需要更新到最新）
