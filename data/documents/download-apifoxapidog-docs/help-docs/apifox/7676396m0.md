# 为什么 Apifox 性能测试中，实际发出的请求数与报告记录数不一致？

这是由 **「优雅关闭（gracePeriodMs）」** 机制引起的预期行为。

当测试时长耗尽后，Apifox 不会立即切断所有连接，而是进入一个宽限期（默认 5000ms）：
* **缓冲作用：** 允许那些已发出但未完成的请求继续执行。
* **强制终止：** 如果宽限期结束请求仍未完成，系统将强制终止它们，以免测试卡死。

为什么会导致数据差异？

两个指标的统计口径不同：

| 统计指标 | 统计范围 | 差异来源 |
| :--- | :--- | :--- |
| **报告记录数** | 最终**成功收到响应**的请求总数。 | 不包含被强制终止的请求。 |
| **实际发出请求数** | 所有**尝试发出**的请求总数。 | **包含**了在宽限期结束后因超时未完成、被强制丢弃的请求。 |
