# 单点登录（SSO）



> [Apifox 商业旗舰版可使用单点登录（SSO）功能](https://www.apifox.com/pricing)

使用 Apifox 时，你可以按照组织为单位配置单点登录（SSO）。Apifox 的单点登录功能支持符合 SAML 2.0 协议的身份提供商（IdP），比如 Microsoft Entra ID（原 Azure Active Directory）。

配置单点登录后，组织内的成员必须定期通过单点登录认证身份才能访问组织内部的资源，从而提高整体的安全性。

与此同时，组织成员可以凭借企业邮箱地址，通过 SSO 直接登录 Apifox 并加入组织。这可以让组织的管理员更方便地邀请成员，同时也简化了组织内成员的登录流程。

## 企业使用单点登录的优点

- **优化用户体验：** 单点登录使得用户只需要记住一个用户名和密码就可以访问所有相关的系统和应用程序。这大大简化了用户的登录过程，提高了用户体验，尤其是对于需要访问多个系统和服务的员工。

- **提高工作效率：** 单点登录使得用户能够快速、无缝地访问必要的资源。这将减少用户用于管理多套账号、密码的时间，将更多时间用于执行关键任务，从而提高生产率。

- **提升安全性：** 单点登录降低了密码疲劳的风险，即用户在不同平台上重复使用相同密码或选择弱密码。有了单点登录，安全措施可以集中管理，从而更容易执行多因素身份验证（MFA）等强大的身份验证策略。

- **简化账户管理：** 单点登录简化了用户管理和权限审核的过程。企业 IT 部门只需要在一个地方管理用户账号和访问权限，减少了管理多个账号和密码的复杂性及可能导致的错误。

- **降低 IT 支持成本：** 用户只使用一套账号、密码，因此由于忘记密码而产生的密码重置请求的数量将会减少，这可以大大减少 IT 支持的工作量和成本。

## 使用 Apifox 单点登录的前提条件

- 身份提供商（IdP）必须支持 SAML 2.0 协议
- 已经在 Apifox 内创建了组织，而且该组织已经购买了商业旗舰版
