# 接口之间如何传递数据


在接口测试中，常常会遇到这样的需求：`B 接口` 的请求参数需要依赖 `A 接口` 返回的数据。为解决这一问题，需要提取 `A 接口` 的返回值并将其传递给 `B 接口`。

以下示例将演示如何提取 `A 接口` 返回的数据中的 `id`，并将其用于 `B 接口` 的请求参数。

## 实现方案

1. **提取数据**：通过 `A 接口` 的后置操作提取返回值，并将数据赋值为变量 *（环境变量、全局变量）*。
2. **引用变量**：在 `B 接口` 的请求参数中使用提取的变量，实现数据传递。

## 示例详解

### 1. 从 A 接口提取变量

在 `A 接口` 的后置操作中，配置如下：
- **变量名称**：`petId`
- **JSONPath 表达式**：`$.data.id` *（用于提取返回数据中 `data` 对象的 `id` 字段）*

完成配置后，发送请求，可以在控制台查看提取到的变量值：

### 2. 在 B 接口中使用变量

在 `B 接口` 的请求参数中，直接引用变量 `{{petId}}`，即可完成数据传递，发送请求后在控制台的 “实际请求” 中可查看引用的变量值：
