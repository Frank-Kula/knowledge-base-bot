# 通过 MCP 使用 Apifox 项目内的 API 文档

Apifox MCP Server 可以将项目内的 API 文档提供给 AI 使用。

## 配置 MCP 客户端

### 前置条件

- 已安装 Node.js 环境（版本号 >= 18，推荐最新的 LTS 版本）
- 任意一个支持 MCP 的 IDE：
  - Cursor
  - VSCode + Cline 插件
  - 其它

<Steps>
  <Step title="在 Apifox 生成 API 个人访问令牌">

   - 创建新的 API 访问令牌，替换下面配置文件中的 `<access-token>`

    <Background>
    
![CleanShot 2025-03-26 at 12.05.49@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/507113/image-preview)

  </Step>
  <Step title="获取 Apifox 项目 ID（可选）">

   - 拿到项目 ID，替换下面配置文件中的 `<project-id>`

### 在 Cursor 中配置 MCP


<Steps>
  <Step title="编辑 MCP 配置文件">
    打开 Cursor 编辑器，点击右上角 「设置」图标，选择左侧「MCP」选项，点击「+ Add new global MCP server」按钮。
      
<Background>

![CleanShot 2025-03-26 at 12.07.53@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/507116/image-preview)
</Background>

  </Step>
  <Step title="添加配置内容">
在打开的 `mcp.json` 文件中添加以下配置，注意替换 `<access-token>` 和 `<project-id>` 为你自己的值：
      
    <Tabs>
      <Tab title="macOS / Linux">
        ```json
        {
          "mcpServers": {
            "API 文档": {
              "command": "npx",
              "args": [
                "-y",
                "apifox-mcp-server@latest",
                "--project=<project-id>"
              ],
              "env": {
                "APIFOX_ACCESS_TOKEN": "<access-token>"
              }
            }
          }
        }
        ```
      </Tab>
      <Tab title="Windows">
        ```json
        {
          "mcpServers": {
            "API 文档": {
              "command": "cmd",
              "args": [
                "/c",
                "npx",
                "-y",
                "apifox-mcp-server@latest",
                "--project=<project-id>"
              ],
              "env": {
                "APIFOX_ACCESS_TOKEN": "<access-token>"
              }
            }
          }
        }
        ```
      </Tab>
    </Tabs>  
  </Step>
  <Step title="验证配置">
    配置完成后，你可以通过向 AI 询问以验证连接是否正常工作（Agent 模式），例如：

```
请通过 MCP 获取 API 文档，并告诉我项目中有几个接口
```
      
如果 AI 能够返回你 Apifox 项目中的 API 信息，说明连接成功。
<Background>

![CleanShot 2025-03-26 at 12.13.11@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/507120/image-preview)
</Background>
  </Step>
      


</Steps>

## 注意事项

- 请将上面的 `<access-token>` 和 `<project-id>` 替换为你个人的 Apifox API 访问令牌和 Apifox 项目 ID。
- 如果需要使用到多个项目的 API 文档，在配置文件里添加配置多个 MCP Server 即可（不同项目有不同的`<project-id>`）。
- 如果团队习惯将 MCP 配置文件同步到代码仓库里，建议删除配置里的"APIFOX_ACCESS_TOKEN": "`<access-token>`"，改成每个成员在自己电脑里配置名为 APIFOX_ACCESS_TOKEN 的环境变量，以避免 APIFOX_ACCESS_TOKEN 泄漏问题。
- 使用私有化部署版本的用户，请在 IDE 的 MCP 的配置文件添加参数："--apifox-api-base-url=`<私有化部署服务器的 API 地址，以 http:// 或 https:// 开头>`"。另外，请确保网络可以正常访问 `www.npm.com`。
      
    ```json
    {
      "mcpServers": {
        "API 文档": {
          "command": "npx",
          "args": [
            "-y",
            "apifox-mcp-server@latest",
            "--project=<project-id>",
            // 私有化部署必须添加以下参数
            "--apifox-api-base-url=<私有化部署服务器的API地址>"
          ],
          "env": {
            "APIFOX_ACCESS_TOKEN": "<access-token>"
          }
        }
      }
    }
    ```

## 常见问题


<Accordion title="Windows 系统配置问题" defaultOpen>

如果在 Windows 操作系统上，前述配置无法正常工作，请使用以下配置：

```json
{
  "mcpServers": {
    "API 文档": {
      "command": "cmd",
      "args": [
        "/c",
        "npx",
        "-y",
        "apifox-mcp-server@latest",
        "--project=<project-id>"
      ],
      "env": {
        "APIFOX_ACCESS_TOKEN": "<access-token>"
      }
    }
  }
}
```
</Accordion>



<Accordion title="Node.js 版本问题" defaultOpen={false}>

如果遇到错误提示 Node.js 版本过低，请确保你安装了 18 或更高版本的 Node.js。你可以通过以下命令检查当前版本：

```bash
node -v
```
</Accordion>


<Accordion title="API 文档更新后，如何让 AI 读取最新数据？" defaultOpen={false}>
API 文档数据会缓存在本地，如果接口文档有更新，记得告诉 AI 刷新接口文档数据，以确保代码生成时使用的是最新版本。

例如：

```js
请重新读取 API 文档数据，在 Pet DTO 里添加 API 文档新增的几个字段
```
</Accordion>
