# 管理迭代分支



在目录树顶端右上角的迭代分支切换功能里点击“管理迭代分支”，或者点击“项目设置 -> 管理迭代分支”，都可进入迭代分支的管理功能页内。可以在此页面内查看本项目中使用的全部迭代分支列表，每个分支的统计数据，也可以对项目的迭代分支进行各种操作。

## 迭代分支的保护

:::tip[]
目前仅支持针对主分支设置保护。
:::

当鼠标 hover 在某个迭代分支上时，可以看到当前分支支持的所有操作。点击“保护”操作，即可针对当前分支进行保护。被保护的分支内容变动操作都是禁用的，需要通过从其它迭代分支中发起合并请求（Merge Request），并被项目管理员审核通过后，才可修改。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/485099/image-preview)
</Background>


如果你的角色是项目管理员，仍然可以直接修改受保护分支的内容，无需通过提交合并请求。如何在其它迭代分支发起合并请求并进行审核，请参考[审核已提交的合并请求](https://docs.apifox.com/merge-sprint-branch.md#审核已提交的合并请求)模块。

## 迭代分支的归档/恢复

针对未被归档的迭代分支，点击归档功能即可将此迭代分支标记为归档状态。归档迭代分支代表此分支已完成它的研发生命周期，后续不再会进行业务上的使用了。所以被归档的迭代分支，在管理迭代分支页中会被折叠，同时在目录树顶端右上角的迭代分支切换功能里，不再显示已归档的迭代分支，不可切换到被归档的迭代分支内。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/485100/image-preview)
</Background>


如果发生一些实际情况，需要查询之前某个已归档迭代分支的内容，并可能选择回溯，则可以在已归档分支中进行恢复操作，让这个分支变为正常使用的分支。变更为正常分支后即可在迭代分支切换功能里切换到这个分支并进行任意操作。


<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/485101/image-preview)
</Background>


## 修改/删除迭代分支

可以点击编辑按钮从而修改迭代分支的名称，或者点击删除按钮彻底删除这个迭代分支。注意，如果确定删除分支，则此迭代分支中的全部内容都会被一并删除并且不可恢复，需要谨慎操作。我们推荐正常情况下多使用“归档”功能而非“删除”。
