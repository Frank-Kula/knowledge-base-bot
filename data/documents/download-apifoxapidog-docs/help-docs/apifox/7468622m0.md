# 解决 ECONNREFUSED: XXXX 错误

`ECONNREFUSED` 错误表示 Apifox 客户端尝试连接目标服务器的指定 IP 地址和端口时，**连接被拒绝了**。



## 错误原因

这个错误通常不是 Apifox 本身的问题，而是出在目标服务器、网络配置或本地设置上。以下是导致 `ECONNREFUSED` 的主要原因：

### 1. 目标服务未运行
你请求的 **目标 API 服务器/服务没有启动** 或在你发送请求时已经崩溃停止了。

### 2. 地址或端口错误
你在 Apifox 请求的 URL 中指定的端口号或 IP 地址/主机名是错误的，服务器没有在那个地址或端口上监听。

### 3. 服务器防火墙阻挡
目标服务器的防火墙（或云服务的安全组）配置阻止了来自 Apifox 客户端的连接。服务器收到了你的连接请求，但直接拒绝了。

### 4. 客户端防火墙阻挡
你本地电脑的防火墙（或安全软件）阻止了 Apifox 应用程序进行外部网络连接。

### 5. 代理配置问题
Apifox 客户端配置了错误的代理设置，导致请求无法正确发送到目标服务器。



## 解决方法

针对上面的原因，你可以按照以下步骤逐一排查和解决：

### 1. 安装插件或使用桌面版

如果你使用的是 Web 版，请安装 [浏览器扩展](https://docs.apifox.com/chrome-extension) 后再发送请求，也可以直接使用 [桌面版](https://docs.apifox.com/download)（推荐）。

### 2. 确认目标服务状态
* **启动/重启服务：** 确保你想要连接的 目标 API 服务或应用，已经在服务器或本地机器上成功启动并稳定运行。

### 3. 核对连接信息
* **检查端口：** 仔细核对 Apifox 请求 URL（例如 `http://127.0.0.1:3000/api/users`）中的端口号 (`3000`)，确保它和你的 API 服务实际监听的端口完全一致。
* **验证地址：** 确保你使用的IP 地址或域名是正确的。如果你使用的是 `localhost` 或 `127.0.0.1`，请确认服务确实运行在你的本地机器上。

### 4. 处理防火墙和安全设置
* **检查服务器防火墙：** 如果 API 部署在远程服务器上，请联系管理员或登录控制台，确认服务器的防火墙或安全组已配置为允许你的 IP 地址或所有地址访问目标端口（例如 80、443、3000 等）。
* **检查本地防火墙：** 尝试临时关闭你本地电脑的防火墙或安全软件。如果问题解决，你需要将 Apifox 添加到防火墙的允许列表中。

### 5. 检查 Apifox 代理设置
