# CLI 运行测试套件

## 选择合适的运行方式

Apifox 提供了多种测试套件运行方式，满足不同的需求：

1. **本地可视化运行**
   - 从本地机器发起，适用于小规模、快速的测试。
   - 适合同时开发和测试，便于实时查看结果和调整。

2. **本地 Apifox CLI 运行**
   - 适合大规模数据或迭代场景，运行速度较快。
   - 支持离线运行，适用于资源有限或不依赖图形界面的情况。

3. **CI/CD 运行**
   - 在 CI/CD 流水线中运行，适用于自动化集成和持续部署流程。
   - 适合频繁运行测试的环境，确保每次代码变更后都能及时验证 API 的稳定性。

4. **自托管 Runner 执行**
   - 团队可以在自己的服务器上部署 Apifox Runner，使用更强大的计算资源来运行测试。
   - 支持定时任务，适用于定期运行场景用例或负载较大的测试需求。


:::tip[]
测试套件中如果使用到了环境/全局变量，那么在选择不同方式运行时，实际使用的变量值可能会不同，导致运行结果不一致。[了解更多](https://docs.apifox.com/run-test-scenarios#%E4%B8%8D%E5%90%8C%E8%BF%90%E8%A1%8C%E6%96%B9%E5%BC%8F%E9%97%B4%E4%BD%BF%E7%94%A8%E4%B8%8D%E5%90%8C%E7%9A%84%E7%8E%AF%E5%A2%83%E5%85%A8%E5%B1%80%E5%8F%98%E9%87%8F%E5%80%BC%E7%9A%84%E8%A7%84%E5%88%99)
:::



## 运行 Apifox CLI

在使用 [Apifox CLI](https://docs.apifox.com/apifox-cli.md) 运行测试套件前，需要先在 Apifox 中创建并编排好测试套件。完成后，即可在命令行中运行测试套件，并获取与图形界面一致的测试结果和测试报告。


<Steps>
  <Step title="进入 CI/CD 配置">
   打开测试套件，切换到 CI/CD 标签页。
  </Step>
  <Step title="配置运行参数">
    在界面中配置运行环境、通知方式等参数。界面中的每一项配置，都会实时反映到下方生成的 CLI 命令中。例如：
      
```js
apifox run --access-token $APIFOX_ACCESS_TOKEN --test-suite 12xx -e 23605xxx -r html,cli
```
      
<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/609715/image-preview)
</Background>

  </Step>
  <Step title="选择 CLI 命令行方式">
    在 CI/CD 平台类型中，选择 CLI 命令行，或者其他平台。
  </Step>
    <Step title="生成访问令牌">
    点击 “添加 Access Token”，并生成 Token，用于 CLI 调用鉴权。[了解更多](https://docs.apifox.com/api-access-token)
        
  </Step>
    <Step title="复制并执行命令">
    点击复制命令，将其粘贴到命令行中执行，即可看到测试执行过程和测试报告。
       
<Background>

![CleanShot 2026-01-04 at 14.27.03@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/609749/image-preview)
</Background>

    
:::tip[]
注意：命令要运行成功，需要先安装 [Apifox CLI](https://docs.apifox.com/install-and-run-cli)，如果已安装有，需要将其更新到最新版本。
:::
  </Step>
</Steps>

## 关于变量与本地配置文件（可选）

在部分场景下，CLI 运行需要依赖本地配置文件：

- **环境 / 全局变量**

    如果选择「导出本地值使用」，需要将导出的变量文件放到 CLI 运行机器中，并通过 `--variables <path>` 指定文件路径。
    
    关于「使用远程值」与「导出本地值使用」的区别，可参考：[不同运行方式下变量使用规则](https://docs.apifox.com/5632037m0#%E4%B8%8D%E5%90%8C%E8%BF%90%E8%A1%8C%E6%96%B9%E5%BC%8F%E9%97%B4%E4%BD%BF%E7%94%A8%E4%B8%8D%E5%90%8C%E7%9A%84%E7%8E%AF%E5%A2%83%E5%85%A8%E5%B1%80%E5%8F%98%E9%87%8F%E5%80%BC%E7%9A%84%E8%A7%84%E5%88%99)
。

- **数据库连接配置**

    如果测试中使用了本地保存的 “数据库连接”，界面会引导导出配置文件，并在 CLI 中通过 `--database-connection <path>` 使用。

    :::tip[]
    建议将 “数据库连接” 信息以变量形式保存，这样在 CLI 运行时无需额外指定 `--database-connection <path>`，也更便于协作和维护。[了解更多](https://docs.apifox.com/database-connections)
    :::

## 命令选项

Apifox CLI 提供了丰富的配置选项来自定义测试集合的运行。详情可以看  [CLI 命令选项](https://docs.apifox.com/cli-command-options.md)。

Apifox CLI 支持和各种流水线工具集成，比如 Jenkins、GitLab、GitHub Actions 等。想了解更多可以看看 [CI/CD 集成](https://docs.apifox.com/cicd.md)。
