# 当发布文档中图片显示“该图片来自其他私有项目,不可访问”,如何处理

报错截图显示如图：


<Background>
![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/567901/image-preview)
</Background>



:::danger[报错原因]
与页面报错提示一致，通常该图片从项目A被复制到项目B，项目B文档发布时，跨项目查看图片，且项目A刚好为非公开，因此图片查看失败
:::

:::info[如何解决]
本质要解决的图片查看权限的问题，
1. 要么项目 A 永远保持公开
2. 要么项目 B 该图片在链接上与本项目的 id 一致，图片重新上传到项目 B
:::
