# 解决 connect EHOSTUNREACH 错误


## 问题现象

在 Apifox 中请求接口时报错：

```
connect EHOSTUNREACH
```

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/610347/image-preview)
</Background>

但使用 Postman 或浏览器请求同一接口可以正常访问。




## 原因说明

当接口地址为内网地址（如 `172.x.x.x`、`192.168.x.x`）时，在 macOS 11 及以上版本中，应用访问本地网络需要系统授权。

如果 Apifox 未被授予「本地网络」访问权限，系统会拦截连接，导致请求失败。


## 解决方法

1. 打开 **System Settings（系统设置）**
2. 进入 **Privacy & Security（隐私与安全）**
3. 点击 **Local Network（本地网络）**

<Background>

![CleanShot 2026-01-05 at 18.05.55@2x.png](https://api.apifox.com/api/v1/projects/5097254/resources/610349/image-preview)
</Background>

5. 找到 **Apifox**，开启访问权限
6. 关闭并重新打开 Apifox，完成后重新发送请求即可。

## 如果仍然无法解决

如果已确认已开启 **本地网络（Local Network）权限**，并重启 Apifox 后仍然报 `connect EHOSTUNREACH`，请继续检查以下项：

### 1. 切换 Agent / 代理

确认 Apifox 当前使用的 Agent 与接口网络环境一致，在请求错误提示中点击 **「更换 Agent」**，选择合适的 Agent 后重新发送请求。



### 2. 确认代理/VPN 配置

如果接口只能通过 VPN 或系统代理访问：

* 确认 Apifox Agent 是否：

  * 使用系统代理
  * 位于 VPN 可访问的网络环境中

必要时可暂时关闭代理或 VPN 进行对比验证。


### 3. 快速自检（可选）

在 Agent 所在环境执行：

```bash
curl http://<接口地址>
```

如果命令行中也无法访问，说明该网络环境本身无法连接该接口。
