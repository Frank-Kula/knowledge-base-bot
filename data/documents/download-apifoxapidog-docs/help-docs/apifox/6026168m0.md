# Apifox 项目、团队和组织的区别

在 Apifox 中，**团队**、**项目**和**组织**是不同层级的概念，分别对应不同的管理和协作范围：

- **组织**：是最大的管理单元，通常代表一个公司或一个大的团队。一个组织可以包含多个团队，适用于公司级别的 API 管理。组织拥有对多个团队的管理权限。

- **团队**：是组织下的一个子单元，通常代表一个小团队或部门。团队负责特定的 API 管理工作，成员可以在团队中协作，进行 API 设计、开发和测试。每个团队有自己的 API 项目和权限管理。

- **项目**：是团队内的具体工作单元，通常用于管理特定的 API 资源。每个项目可以包含不同的 API 文档、接口定义、测试用例等内容，适合在特定的项目中进行集中管理和协作。

简单来说：

- **组织**是最高级别，用于管理多个团队。
- **团队**是组织中的子单元，用于管理团队内具体的 API 项目。
- **项目**是团队中的工作单元，用于管理项目内具体的 API 资源。
