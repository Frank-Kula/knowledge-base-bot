# 如何在 Apifox 在线文档中共享 Header（如 Token）？

**每次在“调试”里调用接口都要手动填 Header，有办法让它自动共享吗？**

可以。推荐使用 “变量” 来实现 Header 的共享。以下是两种常用方式：

### 方法一：在接口的 Header 中直接使用变量

在接口的 Header 参数中这样设置：

```js
Authorization: Bearer {{ACCESS_TOKEN}}
```

用户在在线文档中使用“调试”功能时，只需在变量配置页填写一次 `ACCESS_TOKEN` 的值，所有引用 `{{ACCESS_TOKEN}}` 的接口都会自动带上该值，无需重复填写。

<Background>

![Kapture 2025-07-24 at 10.20.01.gif](https://api.apifox.com/api/v1/projects/5097254/resources/548791/image-preview)
</Background>



### 方法二：在全局参数中设置 Header 并使用变量


<Steps>
  <Step>
    进入 “环境管理 → 全局参数”
  </Step>
  <Step>
    添加一个 Header 类型的参数，例如：

   ```js
   参数名：Authorization
   类型：string
   默认值：Bearer {{ACCESS_TOKEN}}
   ```
  
<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/548792/image-preview)
</Background>
  </Step>
  <Step>
   项目中的接口会默认启用该全局参数，接口中也可以选择是否 “启用” 全局参数。

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/548795/image-preview)
</Background>

  </Step>
    <Step>
    在线文档会自动继承该变量值，实现多个接口间共享。

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/548796/image-preview)
</Background>

  </Step>
</Steps>


无论是在接口 Header 中使用变量，还是通过全局参数引用变量，只要使用了相同的变量名（如 `{{ACCESS_TOKEN}}`），在在线文档中填写一次后即可在多个接口中共享使用。
