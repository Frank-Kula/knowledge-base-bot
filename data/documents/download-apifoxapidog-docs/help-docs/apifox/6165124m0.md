# 为什么测试步骤同步不了接口的参数值？

从接口/接口用例导入的测试步骤，其请求数据可以和关联的接口进行同步。在测试步骤中导入接口/接口用例时可以选择`手动同步`或`自动同步`模式。
#### 选择手动同步
当接口发生变更，需要在测试步骤中点击`同步`按钮,此时会同步接口文档里变动的参数名、Path、Method、前后置操作。
#### 选择自动同步
当接口发生变更，测试步骤会**自动同步**参数名、Path、Method、前后置操作等。


:::caution[注意]
同步的目的是保持接口结构的一致性，而不是更新已有的参数值。在回归测试中，更新参数值可能导致测试用例失效。因此，测试步骤中的原来已经存在的参数值都需要手动更新。
:::
