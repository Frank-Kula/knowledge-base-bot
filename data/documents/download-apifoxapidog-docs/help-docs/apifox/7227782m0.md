# 单接口用例

在 Apifox 版本 `2.7.29` 及以后，在接口详情页中，支持添加用例。

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/622937/image-preview)
</Background>

每一条用例代表对于该接口的一套请求参数值。用户可以有条理对所有用例进行管理，并一键发起运行，完成对于当前接口的测试工作。


:::tip[“用例” 与 “调试用例” 的区别]

- **用例（单接口用例）**
  - 目的是验证接口在各种请求场景下的行为是否符合预期，核心在于全面性。
  - 覆盖正向、反向、边界、安全等多类场景，测试范围广。
  - 相对来说不会经常性修改、频繁运行。
  
- **调试用例**
  - 目的是在开发或问题定位过程中，快速验证接口返回是否符合预期，核心在于快速性。
  - 通常不覆盖完整场景，只聚焦最基础的场景，例如成功、失败。
  - 相对来说会经常性修改、频繁调试。
:::


## 创建用例

你可以通过多种方式进行用例的创建：

### 1. 手动创建

在接口详情的 “用例” 标签页中，可以点击添加用例来手动创建一个用例。

<Background>
 
![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/555455/image-preview)
</Background>

### 2. 从调试用例中导入

可以在添加用例时，选择 “从调试用例添加” 的方式，将现有的调试用例复制或转化成为用例。

- **复制**
    当这条调试用例仍然需要方便大家进行快速调试时，建议使用 “复制” 来将其变为一个用例。
    
- **移动**
    当这条调试用例在一般情况下不会经常被使用来调试，以前就是当做测试当前接口的异常情况来使用时，可以使用 “移动” 来将此用例直接从调试用例转化为用例。这样也方便一些以前已经将用例写在调试用例里的情况，进行快速的迁移。

## 用例详情

一个用例中，包含了以下数据：
- **分组**：基于 “测试目的” 为用例进行的分组，你可以根据测试目的（正向、负向、边界等）来设置用例的分组。
- **名称**：该用例的名称。
- **请求参数值**：涵盖 Path 参数、Query 参数、Header 参数以及 `form-data` 格式的请求体（Body）。
- **请求体内容**：格式可以是 RAW、JSON、XML 等。
- **接口的前后置操作**
- **响应校验设置**：指示校验是否启用，以及指定校验的响应组件。

注意，因为用例一般来说改动频次相对较少，所以用例的内容与关联接口内容是 “手动同步” 的关系。即当接口内容改动（例如增加了一个字段），需要在用例中点击 “立即同步” 按钮，才会将接口中新增与修改的内容同步过来。

## 用例管理

在用例标签页中，你可以针对所有已经创建好的用例进行管理。可以使用左上角的搜索与筛选功能轻松查看到需要查看的用例，也可通过分组标签卡来快速基于分组查看用例。对于具体用例的操作：
- **单个用例**：你可以打开一个新标签页来查看、修改、运行这个用例。基本操作有复制、移动用例的分组、复制这条用例的 cURL、删除用例；
- **多个用例**：当你在列表中选中多个用例，右上角会出现操作按钮，你可以批量对他们进行移动分组、删除、同步接口内容的操作。

## 用例运行

当你查看某条具体的用例时，你可以使用此用例上的 “运行” 按钮，使用当前这一条用例发起请求。请求完成后，这一条用例中就会展示实际响应内容。

<Background>

![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/555488/image-preview)

当然你也可以使用右上角的 “全部运行” 按钮，或选中当前列表中的某些用例再点击此 “运行 x 条” 按钮，来使用全部或特定的一些用例发起请求。发起后会有明显的通话能够查看到当前哪些用例正在请求，以及哪些已经输出了结果。

## 单接口测试的最佳实践

一般来说，单接口的测试会影响到两个角色：后端和测试。这两个角色的分工：
- **后端**：进行接口的代码实际编写与调试；
- **测试**：进行接口的全面覆盖测试与定位问题。

现在 Apifox 已支持用例功能，建议可以通过使用此功能来高效的完成单接口测试工作。使用此功能后的最大亮点：
- 测试可以直接在接口文档中编写与管理单接口的用例；
- 研发可以使用已经写好的用例，自己一键发起测试、查看结果、问题修复，无需再等待测试反馈、低效沟通。

以下是最佳实践具体流程：

<Background>
![image.png](https://api.apifox.com/api/v1/projects/5097254/resources/555494/image-preview)
</Background>




通过此最佳实践流程，可以提升协同效率：
1. 用例在接口文档确认后即可开始编写，并且内容全程同步在 Apifox 中所有项目成员可查看；
2. 研发在调试完成后，可以自己一键测试接口，无需再通知测试并等待测试花费大量时间实际测试与反馈；
3. 研发测试完成可以自己根据详细结果进行问题修复，无需测试在项目管理工具 中写 tickets 再提交给研发修复；
4. 根据实际测试结果来分享和说明问题，不再各说各话难以对齐。
