# Runner 运行环境


## 硬件

| 设备推荐配置 | 部署对象 |
| --- | --- |
| 4 核 4G，30G 磁盘空间 | 自托管 Runner |

1. CPU: 由于 Runner 会并发执行定时任务，所以 CPU 至少 2 核，推荐 4 核或以上。
2. 内存: 由于 Runner 执行定时任务时会加载和生成大量数据，所以内存至少为 4GB，对于更大规模的团队数据需要 8GB 或者更高。
3. 磁盘空间: 考虑到需要存储日志信息，建议至少提供 30GB 的磁盘空间。

## 运行参数

| 参数名 | 参数描述 | 示例值 |
| --- | --- | --- |
| TZ | 配置 Runner 的执行时区，定时任务会根据配置的时区和时间运行，可以参考 [TZ identifier](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones#List) 进行配置。 | America/Los_Angeles

## 网络环境

- Runner 需要与服务端进行通信，运行 Runner 的服务器网络环境至少需要能访问到 Apifox 服务器和支持 Websocket 协议。

- 对于自动化测试的定时任务，请确保 Runner 的服务器网络环境能访问到所有请求的 URL，以便能够正常发起请求；运行完自动化测试后会上传测试报告，请确保服务器网络环境能访问阿里云域名。

- 对于导入数据的定时任务，需要确保运行 Runner 的服务器网络环境能访问到导入数据源的 URL。
