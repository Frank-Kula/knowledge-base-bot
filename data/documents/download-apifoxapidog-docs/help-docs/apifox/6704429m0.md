# 自动化测试中如何让动态值在单次测试中保持一致？

动态表达式（如`{{$person.fullName}}`）每次调用时会重新生成值。如果给变量赋值该动态值表达式，则可能导致测试流程中每次获取数据不一致。
正确做法是：将动态生成的值保存到变量中，后续步骤统一引用该变量而非表达式。变量在赋值时只计算一次，全流程保持不变。

**操作步骤**  
1. **预生成并存储变量**  
   在测试流程的前置脚本中添加：  
   ```javascript
        // 动态值表达式生成一次值
        let realValueString = await pm.variables.replaceInAsync("{{$person.fullName}}");    
        pm.environment.set("fixedPerson", realValueString); // 保存到环境变量
   ```  

2. **引用变量而非表达式**  
   在后续测试步骤中，使用`{{fixedPerson}}`，即可确保全流程使用同一值。
