# Make a request to the endpoint

When you specify an endpoint, a corresponding API endpoint will be automatically generated in the local mock environment. Now you can send requests to this endpoint and receive responses. 

:::tip[]
Learn more about [Mock API data](https://docs.apidog.com/mock-api-data-in-apidog-617869m0.md).
:::

<Steps>
  <Step>
Switch to the **Run** tab.
<Background>
![image.png](https://api.apidog.com/api/v1/projects/544525/resources/344569/image-preview)
</Background>
    </Step>
  <Step>
Select "**Local mock**" as the environment in the upper right corner.
    </Step>
  <Step>
Click **Send** to send the request.
    </Step>
  <Step>
You can view the result of this request in the **Response** area.

    </Step>
   <Step>
Switch to Actual request to view the actual request sent out.
<Background>
    <img src="https://api.apidog.com/api/v1/projects/544525/resources/344586/image-preview" style="width:640px" />
</Background>
    </Step>
  <Step>
Clicking "Save as Case" to the right of the Send button allows you to save this request as an endpoint case, making it convenient for future sending and referencing.
<Background>
    <img src="https://api.apidog.com/api/v1/projects/544525/resources/344571/image-preview" style="width: 500px" />
</Background>
    </Step>
</Steps>

:::tip[]
If you want to send an actual usable API request, you need to put the service URL you want to access to the "Base URL" in the environment management in the upper right corner.

<Background>
    <img src="https://api.apidog.com/api/v1/projects/544525/resources/344570/image-preview" style="width: 640px" />
</Background>
    
Learn more about [Environments](https://docs.apidog.com/environments-variables-in-apidog-577823m0.md).

:::
