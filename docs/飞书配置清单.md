## 📋 待完成配置

### 第 1 步：配置应用权限（5 分钟）

1. 访问飞书开放平台：https://open.feishu.cn/app
2. 找到你的应用"技术支持机器人"
3. 进入"权限管理" → "权限配置"
4. 添加以下权限：

   | 权限名称 | 说明 |
   |---------|------|
   | `contact:user.base:readonly` | 获取用户信息 |
   | `im:message` | 发送和接收消息 |
   | `im:message:group_at_msg` | 接收群@消息 |
   | `bitable:app` | 读写多维表格 |

5. 点击"批量申请权限"

### 第 2 步：获取事件订阅凭证（2 分钟）

1. 在应用中进入"事件订阅"
2. 复制以下信息：

   ```
   Encrypt Key: xxxxxxxx（点击"查看"按钮）
   Verification Token: xxxxxxxx（点击"查看"按钮）
   ```

3. **暂时不要配置请求 URL**（需要先有服务器）
4. 将这两个值添加到 `.env`：

   ```bash
   FEISHU_ENCRYPT_KEY=xxxxxxxx
   FEISHU_VERIFICATION_TOKEN=xxxxxxxx
   ```

### 第 3 步：发布应用（2 分钟）

1. 进入"版本管理与发布"
2. 点击"创建版本"
3. 版本号：`1.0.0`
4. 版本说明：`技术支持机器人初始版本`
5. 点击"保存"
6. 点击"申请发布"
7. 在企业管理后台审核通过

### 第 4 步：创建多维表格（10 分钟）

#### 4.1 创建多维表格应用

1. 在飞书中打开"多维表格"
2. 点击"新建空白表格"
3. 命名为：**技术支持工单**

#### 4.2 创建"缺陷记录"数据表

点击"添加数据表"，创建以下字段：

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 标题 | 单行文本 | 必填 |
| 类型 | 单选 | 选项：Bug/Feature/Usage |
| 优先级 | 单选 | 选项：P0/P1/P2/P3 |
| 版本 | 单行文本 | Apifox 版本号 |
| 环境 | 单行文本 | Web/客户端 |
| 问题描述 | 多行文本 | 详细描述 |
| 复现步骤 | 多行文本 | 1-2-3-4 编号 |
| 预期结果 | 多行文本 | 应该发生什么 |
| 实际结果 | 多行文本 | 实际发生了什么 |
| 是否必现 | 单选 | 选项：必现/偶现 |
| 附件 | 附件 | 截图、日志文件 |
| 提交人 | 人员 | 飞书用户 |
| 创建时间 | 创建时间 | 自动生成 |
| 状态 | 单选 | 选项：待处理/处理中/已解决 |

#### 4.3 创建"需求记录"数据表

再次点击"添加数据表"，创建以下字段：

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 标题 | 单行文本 | 必填 |
| 用户场景 | 多行文本 | 使用场景描述 |
| 当前状态 | 多行文本 | 现有功能限制 |
| 用户期望 | 多行文本 | 期望的解决方案 |
| 用户类型 | 单选 | 选项：个人/企业/私有化 |
| 付费状态 | 单选 | 选项：免费/付费/私有化 |
| 团队规模 | 数字 | 团队人数 |
| 影响程度 | 多行文本 | 业务影响说明 |
| 提交人 | 人员 | 飞书用户 |
| 创建时间 | 创建时间 | 自动生成 |
| 优先级 | 单选 | 选项：P0/P1/P2/P3 |
| 状态 | 单选 | 选项：待评估/已采纳/已拒绝/已实现 |

#### 4.4 获取表格信息

1. 打开创建的多维表格
2. 在浏览器地址栏中找到类似这样的 URL：
   ```
   https://xxx.feishu.cn/base/bascnxxxxxx/app/xxxxxx/table/tblxxxxxx
   ```
3. 复制以下信息：
   - `app_token`：`xxxxxx`（第二个 `/app/` 后面的部分）
   - `table_id`（缺陷表）：`tblxxxxxx`（缺陷表的 ID）
   - `table_id`（需求表）：切换到需求表，复制其 ID

4. 添加到 `.env`：

   ```bash
   FEISHU_APP_TOKEN=xxxxxx
   FEISHU_BUG_TABLE_ID=tblxxxxxx
   FEISHU_FEATURE_TABLE_ID=tblxxxxxx
   ```

### 第 5 步：启动服务器（3 分钟）

#### 方式 A：本地启动（测试用）

```bash
cd D:\document-all\TechnicalSupport\knowledge-base-bot
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload
```

#### 方式 B：使用内网穿透（公网访问）

如果需要配置飞书 Webhook，需要公网地址。使用 ngrok：

1. 下载 ngrok：https://ngrok.com/download
2. 解压并运行：
   ```bash
   ngrok http 8000
   ```
3. 复制显示的公网 URL，例如：
   ```
   https://abc123.ngrok.io
   ```

### 第 6 步：配置 Webhook（5 分钟）

1. 回到飞书开放平台应用
2. 进入"事件订阅"
3. 点击"添加请求"
4. 填写信息：
   - **请求 URL**：`https://abc123.ngrok.io/api/feishu/events`
   - **加密 Key**：已自动填充
   - **验证 Token**：已自动填充
5. 点击"保存"
6. 飞书会发送验证请求，服务器自动响应

### 第 7 步：在测试群中测试（5 分钟）

1. 创建一个测试群聊
2. 在群设置中"添加机器人" → "技术支持机器人"
3. 发送测试消息：
   ```
   接口测试失败了，返回500错误
   ```
4. 观察机器人是否正确回复和收集信息

## 🆘 常见问题

### Q1: 找不到 Encrypt Key 和 Verification Token？

A: 在飞书开放平台 → 应用 → "事件订阅" → 点击"查看"按钮

### Q2: 创建表格时找不到某个字段类型？

A: 某些字段类型可能叫法不同：
- "创建时间" = "记录创建时间"
- "人员" = "成员"或"人员"
- "数字" = "数字"

### Q3: Webhook 验证失败？

A: 确保服务器正在运行，并且 URL 可以从公网访问

### Q4: 消息发送失败？

A: 检查：
1. 应用是否已发布
2. 权限是否已申请并批准
3. 机器人是否已添加到群聊
