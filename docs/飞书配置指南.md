## 第 1 步：创建飞书企业自建应用

### 1.1 访问飞书开放平台

1. 访问：https://open.feishu.cn/app
2. 使用飞书账号登录
3. 点击"创建企业自建应用"

### 1.2 填写应用信息

- **应用名称**：技术支持机器人
- **应用描述**：自动分类技术问题并生成标准化工单
- **应用图标**：上传图标（可选）

### 1.3 获取凭证

创建完成后，在"凭证与基础信息"页面获取：

```
App ID: cli_xxxxxxxxxxxxx
App Secret: ...（需要复制保存）
```

**重要**：将这些信息添加到 `.env` 文件：

```bash
FEISHU_APP_ID=cli_xxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxx
```

## 第 2 步：配置应用权限

### 2.1 开启机器人能力

在"权限管理" → "机器人能力"中开启：

- ✅ 获取与发送单聊消息
- ✅ 获取与发送群聊消息
- ✅ 读取消息和接收事件

### 2.2 配置权限范围

在"权限管理" → "权限配置"中添加以下权限：

| 权限名称 | 权限值 | 用途 |
|---------|--------|------|
| 获取用户信息 | contact:user.base:readonly | 获取提交人信息 |
| 获取群组信息 | chat:group:readonly | 读取群聊信息 |
| 发送消息 | im:message | 发送消息到群聊 |
| 接收事件 | im:message | 接收群聊消息 |
| 多维表格操作 | bitable:app | 读写多维表格 |

### 2.3 发布版本

1. 在"版本管理与发布"中创建版本
2. 填写版本说明
3. 点击"发布"
4. 在企业管理后台审核通过

## 第 3 步：创建多维表格

### 3.1 创建多维表格应用

1. 在飞书中打开"多维表格"
2. 点击"新建表格"
3. 命名为"技术支持工单"

### 3.2 创建缺陷记录表

**表名**：缺陷记录（Bug）

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 标题 | 单行文本 | 问题标题 |
| 类型 | 单选 | Bug/Feature/Usage |
| 优先级 | 单选 | P0/P1/P2/P3 |
| 版本 | 单行文本 | Apifox 版本号 |
| 环境 | 单行文本 | Web/客户端 |
| 问题描述 | 多行文本 | 详细描述 |
| 复现步骤 | 多行文本 | 步骤编号 |
| 预期结果 | 多行文本 | 应该发生什么 |
| 实际结果 | 多行文本 | 实际发生了什么 |
| 是否必现 | 单选 | 必现/偶现 |
| 附件 | 附件 | 截图、日志 |
| 提交人 | 人员 | 飞书用户 |
| 创建时间 | 日期 | 自动生成 |
| 状态 | 单选 | 待处理/处理中/已解决 |

### 3.3 创建需求记录表

**表名**：需求记录（Feature）

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 标题 | 单行文本 | 需求标题 |
| 用户场景 | 多行文本 | 使用场景描述 |
| 当前状态 | 多行文本 | 现有功能限制 |
| 用户期望 | 多行文本 | 期望的解决方案 |
| 用户类型 | 单选 | 个人/企业/私有化 |
| 付费状态 | 单选 | 免费/付费/私有化 |
| 团队规模 | 数字 | 团队人数 |
| 影响程度 | 多行文本 | 业务影响 |
| 提交人 | 人员 | 飞书用户 |
| 创建时间 | 日期 | 自动生成 |
| 优先级 | 单选 | P0/P1/P2/P3 |
| 状态 | 单选 | 待评估/已采纳/已拒绝/已实现 |

### 3.4 获取表格信息

1. 打开多维表格
2. 在浏览器地址栏获取：
   ```
   https://example.feishu.cn/base/xxxxxx/app/xxxxxx/table/xxxxxx
   ```
   - `app_token`：xxxxxx（第二个）
   - `table_id`：xxxxxx（第三个）

添加到 `.env`：
```bash
FEISHU_APP_TOKEN=xxxxxx
FEISHU_BUG_TABLE_ID=xxxxxx
FEISHU_FEATURE_TABLE_ID=xxxxxx
```

## 第 4 步：配置事件订阅

### 4.1 获取 Encrypt Key 和 Verification Token

1. 在开放平台应用中，进入"事件订阅"
2. 复制以下信息：
   ```
   Encrypt Key: xxxxxxxx
   Verification Token: xxxxxxxx
   ```

添加到 `.env`：
```bash
FEISHU_ENCRYPT_KEY=xxxxxxxx
FEISHU_VERIFICATION_TOKEN=xxxxxxxx
```

### 4.2 配置请求 URL

**注意**：需要先有公网服务器才能配置 Webhook。

如果暂时没有服务器，可以先使用内网穿透工具：
- ngrok: https://ngrok.com/
- frp: https://github.com/fatedier/frp
- localtunnel: https://localtunnel.github.io/www/

**URL 格式**：
```
https://your-domain.com/api/feishu/events
```

### 4.3 验证请求 URL

飞书会发送验证请求，服务器需要正确响应（详见代码实现）。

### 5.1 在测试群中添加机器人

## 🚀 快速开始

准备好后，运行以下命令启动服务器：

```bash
# 安装依赖
pip install fastapi uvicorn feishu-bitable

# 启动服务器
python -m uvicorn src.main:app --host 0.0.0.0 --port 8000
```
